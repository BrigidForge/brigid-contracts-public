# Versioning

This repository should publish contract snapshots intentionally rather than acting as a rolling mirror of day-to-day development.

## Recommended Practice

- use tags for public release points
- map each tag to a public deployment record
- update this repository when a contract set is ready for external reference

## Example

- `v1.0.0-public`
  - first public contract bundle
- `v1.1.0-public`
  - updated public bundle aligned with a later verified deployment set
